# Функции и Модули

## Создание собственных функций
Если у вас есть код, который нужно повторять несколько раз, вы можете выделить его в функцию.
Для создания функции используется слово `функция`, за которым следует её имя и параметры в скобках. Возврат значения осуществляется словом `вернуть`.

```son
функция сложение(а, б):
    пусть результат = а + б
    вернуть результат
конец

пусть сумма = сложение(5, 7)
напечатай "5 + 7 = " + строка(сумма)
```

Вы также можете создавать функции, которые ничего не возвращают, а просто выполняют действие:

```son
функция поприветствовать(имя):
    напечатай "Привет, " + имя + "!"
конец

поприветствовать("Алексей")
```

## Использование модулей
Модули — это встроенные библиотеки, которые предоставляют доступ к расширенному функционалу (графика, игры, файловая система, математика).
Чтобы использовать модуль, его нужно объявить в самом начале файла с помощью слова `модуль`.

```son
модуль время
модуль математика

напечатай "Сейчас будет случайное число:"
время.пауза(1000) # Ждем 1 секунду
напечатай математика.случайное(1, 100)
```

### Доступные модули:
В языке SonLang встроены следующие мощные модули (каждый из них описан в отдельном файле документации):
* `математика` — продвинутые вычисления (случайные числа, синусы, корни).
* `графика` — создание окон и рисование геометрических фигур.
* `gui` — создание графического интерфейса пользователя (кнопки, текст).
* `игра` — хелперы для разработки игр (столкновения).
* `систем` — работа с Windows/Linux (вызов консольных команд, файлы, переменные).
* `время` — паузы и измерение времени.
* `файл` — чтение и запись текстовых файлов.
* `звук` — проигрывание системных и внешних звуков.

## 🛠 Создание собственных (пользовательских) модулей

Вы можете создавать собственные библиотеки и модули прямо на языке SonLang! Это позволяет вынести повторяющийся красивый дизайн или сложную логику в отдельные файлы, чтобы не перегружать основной код.

### 1. Напишите файл модуля
Создайте файл (например, `дизайн.son`) и напишите в нем функции. Вы также можете импортировать в него стандартные модули, такие как `графика` или `gui`:

```son
# Файл: дизайн.son
модуль графика
модуль gui

# Функция рисует кнопку с красивой объемной тенью
функция кнопка_премиум(id, х, у, ш, в, текст, цвет):
    # Тень
    графика.прямоугольник(х + 3, у + 3, ш, в, "#111122")
    # Сама интерактивная кнопка поверх тени
    gui.кнопка(id, х, у, ш, в, текст, цвет)
конец

# Функция рисует заголовок с разделительной линией
функция заголовок(х, у, текст, цвет):
    gui.метка(х, у, текст, цвет)
    графика.линия(х, у + 18, х + 200, у + 18, цвет)
конец
```

### 2. Импортируйте и используйте его в основной программе
Чтобы использовать этот модуль в вашей игре или программе, просто напишите `модуль дизайн` (название файла без расширения `.son`). Интерпретатор сам найдет этот файл в текущей папке, выполнит его и сделает все его функции доступными через точку!

```son
# Файл: игра.son
модуль графика
модуль gui
модуль время

# Импортируем наш собственный модуль из файла дизайн.son!
модуль дизайн

графика.окно(400, 300, "Тест модулей", истина)
gui.инициализировать()

пока графика.закрыто() == ложь:
    графика.очистить("#1A1A2E")
    
    # Вызываем наши собственные функции через точку!
    дизайн.заголовок(20, 20, "ГЛАВНОЕ МЕНЮ", "#00FF88")
    дизайн.кнопка_премиум("старт", 50, 100, 300, 50, "НАЧАТЬ ИГРУ", "#E94560")
    
    gui.нарисовать()
    графика.обновить()
    
    если gui.нажата("старт"):
        напечатай "Игра запускается..."
    конец
    
    время.пауза(16)
конец
```

Таким образом, любой человек может написать огромную библиотеку с красивыми элементами интерфейса (тенями, градиентами, анимациями) и поделиться ею с другими программистами!
