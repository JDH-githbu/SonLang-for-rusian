# Лучшие Практики и Чистый Код

Написание понятного, структурированного кода — залог успеха любого программиста. В этом документе собраны рекомендации по оформлению программ на языке SonLang.

---

## 1. Стиль Именования (Naming Conventions)

Поскольку SonLang полностью поддерживает кириллицу, пишите имена переменных и функций на русском языке. Это делает код живым и легко читаемым.

*   **Переменные и функции**: Используйте строчные буквы и разделяйте слова нижним подчеркиванием (`snake_case`).
    *   *Хорошо:* `пусть скорость_игрока = 5`, `функция нарисовать_сетку():`
    *   *Плохо:* `пусть СкоростьИгрока = 5`, `пусть СКОРОСТЬ = 5`
*   **Имена модулей**: Всегда пишите маленькими буквами в единственном числе: `модуль графика`, `модуль игра`.

---

## 2. Организация Файла Программы

Чтобы ваш код не превращался в "спагетти", придерживайтесь стандартной структуры файла:

```son
# ==========================================
# 1. ПОДКЛЮЧЕНИЕ МОДУЛЕЙ И ИМПОРТЫ
# ==========================================
модуль графика
модуль gui
модуль время
импорт дизайн

# ==========================================
# 2. ГЛОБАЛЬНЫЕ НАСТРОЙКИ (КОНСТАНТЫ)
# ==========================================
пусть ШИРИНА = 400
пусть ВЫСОТА = 300

# ==========================================
# 3. СОСТОЯНИЕ ИГРЫ / ПЕРЕМЕННЫЕ
# ==========================================
пусть счетчик = 0
пусть жив = истина

# ==========================================
# 4. ИНИЦИАЛИЗАЦИЯ ДВИЖКОВ
# ==========================================
графика.окно(ШИРИНА, ВЫСОТА, "Моя игра")
gui.инициализировать()

# ==========================================
# 5. ГЛАВНЫЙ ИГРОВОЙ ЦИКЛ
# ==========================================
пока графика.закрыто() == ложь:
    графика.очистить("черный")
    
    # ... логика и отрисовка ...
    
    графика.обновить()
    время.пауза(16)
конец
```

---

## 3. Избегайте Засорения Окружения (Shadowing)

В SonLang переменные внутри функций могут скрывать глобальные переменные, если они объявлены через `пусть`.
*   Если вы хотите изменить **глобальную** переменную внутри функции, **не** пишите ключевое слово `пусть` внутри функции.
*   **Пример:**
    ```son
    пусть счет = 0
    
    функция заработать_очко():
        счет = счет + 1 # Правильно: изменяет глобальный счет
    конец
    
    функция сломать_счет():
        пусть счет = 99 # Ошибка! Создает локальный счет, глобальный остается 0
    конец
    ```

## 4. Оптимизация Отрисовки (Производительность)

При работе с модулем `графика` помните:
1. **Не создавайте окна (`графика.окно`) внутри цикла**. Окно создается ровно один раз перед главным циклом `пока`.
2. **Всегда вызывайте `графика.очистить`** в самом начале кадра, чтобы стереть картинку предыдущего кадра.
3. **Всегда вызывайте `графика.обновить`** в самом конце кадра. Без этого экран останется пустым.
4. **Добавляйте `время.пауза(16)`** в конец цикла. Это разгрузит процессор вашего ПК и установит стабильные 60 кадров в секунду. Без паузы игра будет использовать 100% мощности процессора, пытаясь выдать тысячи кадров в секунду.
