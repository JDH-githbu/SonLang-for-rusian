# Физика и Математика в 2D Играх

Разработка игр — один из лучших способов изучения программирования. В этом документе мы разберем математические и графические основы, которые помогут вам создавать игры на SonLang.

---

## 1. Экранная система координат

В компьютерной графике (в отличие от школьной геометрии) точка начала координат `(0, 0)` находится в **левом верхнем углу** экрана.

```
(0,0) ──────────────────────────► X (ширина)
  │
  │
  │     ● (x=100, y=50)
  │
  ▼
  Y (высота)
```

* **Ось X** направлена слева направо. При увеличении X объект движется **вправо**.
* **Ось Y** направлена **сверху вниз**. При увеличении Y объект движется **вниз**.

---

## 2. Логика Движения и Вектор Скорости

В играх движение объектов обновляется каждый кадр внутри бесконечного цикла. Мы управляем объектом с помощью смещения координат:

```son
пусть х = 200
пусть у = 200
пусть скорость_х = 5
пусть скорость_у = 2

пока графика.закрыто() == ложь:
    графика.очистить("черный")
    
    # Сдвигаем координаты на вектор скорости каждый кадр
    х = х + скорость_х
    у = у + скорость_у
    
    # Рисуем объект в новых координатах
    графика.круг(х, у, 15, "красный")
    
    графика.обновить()
    время.пауза(16)
конец
```

Если скорость по оси X (`скорость_х`) положительная, объект летит вправо. Если отрицательная — влево.

---

## 3. Отскок от Стен (Простые Столкновения)

Чтобы объект не улетал за пределы экрана, мы должны проверять его координаты на каждом кадре и менять направление скорости на противоположное:

```son
# Отскок по горизонтали (ширина окна = 400)
если х < 0 или х > 400:
    скорость_х = -скорость_х # Меняем направление скорости на противоположное
конец

# Отскок по вертикали (высота окна = 300)
если у < 0 или у > 300:
    скорость_у = -скорость_у
конец
```

## 4. Сетка и Шаг в стиле "Змейка"

В таких играх, как Змейка или Тетрис, объекты двигаются не плавно попиксельно, а строго по невидимой сетке.
* Для этого мы задаем `размер_клетки` (например, 20 пикселей).
* Все координаты спавна и движения должны быть кратны размеру клетки:
```son
пусть х = х + (направление_х * размер_клетки)
```

Функция `игра.случайное` отлично подходит для генерации координат на сетке:
```son
# Случайная клетка от 1 до 19 по горизонтали
пусть еда_х = игра.случайное(1, 19) * размер_клетки
```

## 5. Обнаружение Столкновений (AABB Collision)

Самый популярный и быстрый метод проверки столкновений — **AABB (Axis-Aligned Bounding Box)**. Он проверяет пересечение двух не повернутых прямоугольников.
Этот алгоритм уже встроен в SonLang как функция `игра.столкновение`. Она проверяет, перекрываются ли диапазоны координат двух объектов по осям X и Y.

Формула, работающая под капотом `игра.столкновение`:
```
Пересечение по X:  (x1 < x2 + w2) и (x1 + w1 > x2)
Пересечение по Y:  (y1 < y2 + w2) и (y1 + w1 > y2)
```
Если оба условия выполняются одновременно, объекты столкнулись! Используйте ее для любых проверок попадания пуль, сбора монет или препятствий.
